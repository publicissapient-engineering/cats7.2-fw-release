    DYNAMIC PROPERTY

    Variables used in the DYNAMIC PROPERTY column in the OR sheet must be either OR Variables (which is defeined using SetORVarirable ) or Global Variable (defined using SetVariable, SetVariableValue, SetAttributeVariable, SetCSSVariable)

    If the variable used in the Dynamic Property is not set or initialized then the dynamic property will not be used, instead the property values will be picked from the standard property columns (XPATH, CLASSNAME, HTMLID,LINKTEXT, TAGNAME, NAME, PARTIAL LINKTEXT).

    If the OR Variable is used in Dynamic Property then user has option to reset the variable which is as good as the variable is not initialized. Once the ResetORVariable is called the object will start using the default properties columns to identify the objects. User will not have the ablity to RESET the Global Variables used in the Dynamic Property however he can update the variable value.

    Dynamic Property supports three properties xpath, htmlid and classname only.

    Syntax: propertyName::[Global Variable] or [OR Variable] or [Expression with Global or OR Variable]. Refer the below tables

    Known Issue: XPath containing axisname and nodetest is not working as it uses the delimiter ::     OR Example:
    SL NO. PAGE ELEMENT NAME XPATH DYNAMIC PROPERTY
    Eg-1 PDP productName //div[@class='prd']//span[1] xpath:://div[@class='prd']//span [@text()='{{ItemName}}']
    Eg-2 PDP swatchColor //div[@class='prd']//span[1] xpath:://div[@class='prd']//span[@text()='{{ItemName}}'] /../img[@title='{{{color}}}']
    Eg-3 Checkout continueCheckout //input[@name='submit_checkout'] htmlid::{{{checkoutbuttonid}}}

    SL NO. PAGE ELEMENT NAME HTMLID DYNAMIC PROPERTY
    Eg-4 ShoppingCart AddToCart btn_addCart xpath:://input[@name='{{buttonName}}']
    Eg-5 Checkout orderNumber xpath:://a[text()='{{orderNumber}}]']; htmlid::ordernum_{{orderNumber}}_lnk; classname::{{order_confirmation}}
    Eg-6 Checkout orderNumber tagname::{{htmlTag}}; name::order_{{name}}; linktext::{{order_confirmation}}


    Explained Eg-1: Object productName has static xpath and dynamic property with single OR variable. User has to call SetORVarirable to define the variable value productName. If the variable used in the dynamic property is not initialized then default xpath will be used to identify the object

    Explained Eg-2: Object productName has static xpath and dynamic property with two variables (one is OR Variable {{ItemName}} and another is globale variable {{{color}}}). Dynamic Property will be ignored if any of the used variables are not initialized

    Explained Eg-3: Object continueCheckout will use html id from the dynamic property for object identification if OR Variable is set, otherwise it will use the static xpath

    Explained Eg-4: Object AddToCart has static html id and dynamic property (xpath) with one variable. This will not be helpful as the html id got higher precedence than xpath. However xpath will be updated by dynamic property based on the OR Variable set during runtime for the object AddToCart

    Explained Eg-5: Dynamic property can have more more than one property at the same time, this will just update the runtime property and will be used. However property used to identify the object will be as per the framework hierachy. The sequence defined in the Dynamic Property has no role.

    CREATIVE DISTANCE MEASURE

    Distance between two elements can be verified using different mode, to alter the distance calculation user needs to mention the type of distance to be calculated in the test data. These are applicable to applicable to following creative actions VerifyLayoutLeft, VerifyLayoutRight, VerifyLayoutBelow, VerifyLayoutAbove

    Test Data margin: distance between two elements will be calculated from their margins
    border: distance between two elements will be calculated from their borders excluding margin
    padding: distance between two elements will be calculated from their padding excluding margin and borders
    actual: distance between two elements will be calculated from their actual content excluding margin, borders and padding
    default: when the test data is left blank distance between two elements will be calculated from their borders excluding margins
